Basic Operations

Convolution Operation

Each Convolution Filter consists of (kernel_size * kernel_size * num_channels), so the output size is determined by the number of filters in the given layer.

Also note that there‚Äôs a notion of ‚Äústride‚Äù and ‚Äúpadding‚Äù.

Having padding = 1 means the original input will be padded with 1 unit of ‚Äú0‚Äùs on each side, so assume that the original image is w*h*c, the padded image would be (w+2)*(h+2)*c

Paddings:

  1. Same ‚áí Produces an output that matches w*h of the input, regardless of the kernel size of the filters
  1. Valid ‚áí No Padding

One by one convolution(pointwise convolution)

A way to control feature numbers

think about a convolution layer n filters of a 1√ó11 \times 11√ó1Ôªø kernel size with a stride of (1,1)(1,1)(1,1)Ôªø. This would mean that the layer preserves w*h, but changes c ‚áí n

Pooling Layers(Max Pooling / Average Pooling)

Similar to Convlution layer, the pooling layers also have a parameter on kernel size, stride, and padding(although padding is usually not used).

The Pooling Layers, instead of doing a sum on each of the operated cells, does different things

  1. Max Pooling simply pulls out the max value in those cells
  1. Average Pooling computes the average value of those cells

The intuition behind this might be:

  1. Pooling layers extracts features from a huge input and puts it into a lower-dimension output(shrinks w*h, not including c)
    1. Think about a ‚Äúcat-or-not‚Äù recognizer, maybe the network doesn‚Äôt care where the cat mustache is, but as long as there is a cat mustache there‚Äôs a cat, then ‚Äúmax-pooling‚Äù can extract the mustache into a smaller output matrix.

Depthwise Convolution(MobileNet)

In a depthwise convolution, the number of filters is locked on to the # of channels of the input, and each filter is applied to its designated input channel. So depthwise seperable convolution does not channge the number of channels, ncn_cnc‚ÄãÔªø, from input to output.

It dramatically decreases multiplications needed for each convolution operation (but of course, there‚Äôs a cost to it - performance)

Transposed Convolution(U-Net)

With U-Net, one major question is with such big number of outputs, how do you go from a small set of inputs to a large set of outputs?

  1. Padding is now instead put on the output layer(those ignored pads will be disgarded when outputting)
  1. Filters, instead of convoluting and applying on each batch of input cells, is multiplied a single input cell and applied into the ‚Äúconvoluting subset‚Äù on the output image
  1. In terms of dealing with multiple input/output channels (ncn_cnc‚ÄãÔªø), the transposed convolution acts the same as normal convolution, as it assigns a kh√ókwk_h \times k_wkh‚Äã√ókw‚ÄãÔªø tensor (kh=kernel_size2,kw=nc_inputk_h = kernel\_size^2, k_w = n_{c\_input}kh‚Äã=kernel_size2,kw‚Äã=nc_input‚ÄãÔªø) for each filter.

Object Detection

Sliding Window Algorithm

Repeatedly applying a small block filter to check if an image ‚Äúcontains‚Äù object, and if it does, we know there‚Äôs a object in the specific bounding box.

  1. Very Computationally Expensive(think about how many iterations of classification neural networks we have to go through for us to do this kind of object detection)
  1. Not really accurate prediction

Convolution Implemention of sliding window

Since a lof of the computations in sliding window is redundant, we can simply apply convolution layers on the ‚Äúenlarged‚Äù, or ‚Äúfull‚Äù image

Idea of FC(Fully Connected) layer can be thought of a 1*1 kernel conv layer is pretty important here so that we can perform those FC layer on a ‚Äúbigger‚Äù input from the previous layers.

Expanding on the definition of Anchor Boxes

Anchor Boxes allows each cell to detect multiple objects...

Anchor boxes are a set of predefined bounding boxes of a certain height and width. These boxes are defined to capture the scale and aspect ratio of specific object classes you want to detect and are typically chosen based on object sizes in your training datasets.

The network does not directly predict bounding boxes, but rather predicts the probabilities and refinements that correspond to the tiled anchor boxes.

Concept of IOU(Intersection over Union)

IOU(A,B)=A‚à©BA‚à™BIOU(A,B) = {A \cap B \over A \cup B}IOU(A,B)=A‚à™BA‚à©B‚ÄãÔªø

Computes the fraction of overlap between A and B

This concept becomes very handy when doing non-max supression.

Non-max Supression

Sometimes a YOLO model can predict an object with overlapping bounding boxes, and non-max supression helps you to gain a much clearer prediction result.

Steps:

  1. Suppress ALL boxes with pc‚â§thresholdp_c \le thresholdpc‚Äã‚â§thresholdÔªø
    1. We want GOOD predictions
  1. Pick max and supress non-max
    1. First highlight the bounding box with the max probability
    1. Find the IOU of this bounding box with other bounding boxes (with the same class)
      1. If the iou‚â•thresholdoverlapiou \ge threshold_{overlap}iou‚â•thresholdoverlap‚ÄãÔªø, then eliminate the other bounding box
      1. If not, then leave the other bounding box
    1. Repeat the first step on the second max probability, third...until we found all rectangle to have either been passed through or eliminated out.

R-CNN(CNN with region proposals)

stands for Regions with Convolutional Neural Nets

The downside of Convolutional Sliding Window is you spend some time trying to detect an object from a sub-part of the image that clearly doesn‚Äôt have an object, say this one...

So instead of using a sliding window, let an algorithm(With R-CNN, its called a segmentation algorithm) propose a few regions and we just run our classifier on those few regions.

After proposing, R-CNN output a label and a bounding box. Note that the bobunding box can but doesn‚Äôt necessary have to be the bounding box the segmentation algorithm proposes, it actually trys to figure out a better bounding box.

Turns out that this is still slow...So work has been done to try to speed up this algorithm
